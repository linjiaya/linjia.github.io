vue注意事项

发表于 2018-03-22

安装启动问题

vue init webpack “yourProject name”时，出现以下提示

This will install Vue 2.x version of the template.

For Vue 1.x use: vue init webpack#1.0 firstVue

vue-cli · Failed to download repo vuejs-templates/webpack: unable to verify he first certificate

意思是：

即将要安装vue2.x版本的模板

如果要使用vue1.x 请使用命令vue init webpack#1.0 firstVue

下载vuejs模板失败，出现这个问题，应该是npm不稳定造成的，请跟换为淘宝镜像cnpm

使用以下命令安装cnpm
$ npm install -g cnpm --registry=https://registry.npm.taobao.org

安装好了以后，请直接使用以下命令重新安装下vue-cli
$ cnpm install -g vue-cli

重新再执行以下命令试试
$ vue init webpack firstVue

安装的vue是可以选择是否安装vue-router的，会自动建好router文件夹

bootstrap的引入

首先下载bootstrap
npm i bootstrap --save
再在main.js中写
import 'bootstrap/dist/css/bootstrap.min.css'
import 'bootstrap/dist/js/bootstrap.min.js'

less的引入

安装less依赖
npm install less less-loader --save
修改webpack.config.js文件，配置loader加载依赖，让其支持外部的less,在原来的代码上添加

现在基本上已经安装完成了，然后在使用的时候在style标签里加上lang=”less”里面就可以写less的代码了(style标签里加上 scoped 为只在此作用域 有效)（或者@import ‘./index.less’; //引入全局less文件）。

页面渲染问题

一般项目都是各种组件嵌套，如果再父组件中dom元素会改变的画会导致页面重绘。

目前我的解决办法是：所有的组件都分开，变成各自独立的部分，这不ß就是vue的特点么。在父组件中避免有dom的改变。要知道所有对dom的操作都是很昂贵的。

路径问题

vue因为有vue-cli，在js中的url会进行vue-load处理，不能够直接写url，需要加上require(在这里写上你的url)。
