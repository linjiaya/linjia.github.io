vue生命周期

发表于 2019-07-19

vue的生命周期，官网po了一张生命周期的图示，然后，我相信你会回来的。

每个组件都和生命周期有关，每个组件啥时候该做什么？该有什么？然后下一步要干嘛？怎样会触发？触发了然后会怎样？下面就来唠一唠。

那我也先po一张生命周期图先🤪

生命周期有8个，分别是：

创建=>挂载=>更新=>销毁

创建：beforeCreate、create

挂载：beforeMounted、mounted

更新：beforeUpdate、updated

销毁：beforeDestroy、destroyed

我是用增删改查来的场景来理解这个过程

好的，现在我开始 new Vue() 了，初始化一下init 事件和生命周期

这个时候呢，就是你已经填好数据了，也调了新增的接口，后台也保存了你的数据。前端得到后台提示新增成功！好！这个时候我们前端知道，哦，我刚新建了一个东西(data)并且成功了，但是我也不知道他长啥样（$el）。这样就新增完成啦！

根据你提供的数据(data)，那他就要去造东西啦。咋造呢？

  • el属性对生命周期的影响

∴ 如果没写el的话记得写$mount(el);

然后看有没有template

  • template属性对生命周期的影响
    1. 在实例内部有template属性的时候，直接用内部的，然后调用render函数去渲染。
    2. 在实例内部没有找到template，就调用外部的html。实例内部的template属性比外部的优先级高。
    3. 要是前两者都不满足，那么就抛出错误。

mounted

el： ✔︎ （编译完成后的样子）<div>123</div>

$el： ✔︎

data：✔︎

这个时候算是一个vue从初始化到展现到你面前的流程都走完了。

这个时候如果要取dom对象，走到这个时候才能拿到的，比如要获取一个canvas。

更新（修改）

beforeUpdate

这个过程中，我们会发现，当一个数据发生改变时，你的视图也将随之改变，整个更新的过程是：数据改变——导致虚拟DOM的改变——调用这两个生命钩子去改变视图

重点：这个数据只有和模版中的数据绑定了才会发生更新。

比如有个组织区域我要改一下，我把组织从A → B。在下拉框选择了B，如果模板上面有用到这个数据（一定是模板上要用到了这个数据，没有也不会触发）发生了改变了，就会触发beforeUpdate。

好，数据改变，那就到更新虚拟dom了，执行render函数

然后就是Updated了

如果在beforeUpdate 和 Updated 执行的时候回，我又改了。会怎样执行？

会把第一回合的更改执行完后，再更改再走一遍beforeUpdate → render → updated。走到没有改动为止。

可想而知，我都以为你改完了，你特么还改，那我就一直改，改到死。然后它就死了。

所以没事别在beforeUpdate和updated里面改值

销毁（删除）

beforeDestory

这里呢好比点击了删除，然后有个提示问我要删除了啊？是不是想问为啥不提示我“你要不要删？”，不存在的，你都过来了，我默认你要删了，可没有什么后悔药可以吃的。

页面表现就是离开这个组件时触发的，不是看不到（前端应该知道区别的吧），不知道我再说下，就是我隐身站在你旁边，和我不在你身边的意思。

在这里可以做一些事件触发之类的事情，比如离开的时候要让下一个要来的人知道你想告诉他什么

你们能够同时存在的时机就是从现在开始，在这你可以告诉他要干嘛干嘛

怎么告诉呢，比如eventbus。

之后呢，就开始拆除数据的监听，子组件，事件监听。把我这辈子的事迹都拆下
